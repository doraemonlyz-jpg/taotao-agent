Question 1 · live coding

从 0 写一个 ReAct agent · 不用框架

最常考的开场题 · 用来验证你"是真懂 agent loop 不是只会调 LangChain"

难度 mid 限时 60-90 min 形式 live coding · 共享屏幕 面方 字节 / 阿里 / 创业公司常见

用 Python 实现一个最小 ReAct agent（不用 LangChain / LangGraph）。给定 2 个 tool（search 和 calculator），让它能回答 "What is the population of the country whose capital is Tokyo, divided by 1000?" 这种 multi-hop 问题。

§1.1参考实现 (~120 行 · 30 min 写完)

# react_agent.py
import json
from openai import OpenAI
from typing import Callable

client = OpenAI()

# ---------------- tools ----------------
def search(query: str) -> str:
    """Mock web search."""
    fake_db = {
        "capital of japan": "Tokyo",
        "population of japan": "125,000,000",
        "capital of france": "Paris",
        "population of france": "68,000,000",
    }
    for k, v in fake_db.items():
        if k in query.lower():
            return v
    return "no result"

def calculator(expression: str) -> str:
    """Eval a math expression like '125000000 / 1000'."""
    try:
        return str(eval(expression, {"__builtins__": {}}, {}))
    except Exception as e:
        return f"error: {e}"

TOOLS: dict[str, Callable] = {"search": search, "calculator": calculator}

# ---------------- prompt ----------------
SYSTEM = """You are a research agent. Reason step-by-step (ReAct).
At each step output exactly ONE of:

  Thought: <your reasoning>
  Action: <tool_name>
  Action Input: <input>

OR when done:

  Thought: <your reasoning>
  Final Answer: <final answer>

Available tools:
- search(query): web search returning a string
- calculator(expression): eval a python math expression

Always start with 'Thought:'. Never combine multiple actions in one turn.
"""

# ---------------- parser ----------------
def parse_step(text: str) -> dict:
    """Parse LLM output into {thought, action, action_input} or {thought, final}."""
    lines = [l.strip() for l in text.strip().split("\n") if l.strip()]
    out = {}
    for line in lines:
        if line.startswith("Thought:"):
            out["thought"] = line[len("Thought:"):].strip()
        elif line.startswith("Action:"):
            out["action"] = line[len("Action:"):].strip()
        elif line.startswith("Action Input:"):
            out["action_input"] = line[len("Action Input:"):].strip()
        elif line.startswith("Final Answer:"):
            out["final"] = line[len("Final Answer:"):].strip()
    return out

# ---------------- agent loop ----------------
def run_agent(question: str, max_steps: int = 10) -> str:
    history = SYSTEM + "\n\nQuestion: " + question + "\n"
    seen_actions = set()  # bug防护: 重复 action 检测

    for step in range(max_steps):
        resp = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": history}],
            stop=["\nObservation:"],  # 关键: 让 LLM 在 Action Input 后停
            temperature=0,
        )
        text = resp.choices[0].message.content
        history += text + "\n"
        parsed = parse_step(text)

        if "final" in parsed:
            return parsed["final"]

        action = parsed.get("action")
        action_input = parsed.get("action_input", "")
        if not action:
            history += "Observation: malformed step. Use 'Action:' or 'Final Answer:'.\n"
            continue

        # bug防护: 同一 action+input 重复 3 次直接终止
        sig = (action, action_input)
        if sig in seen_actions:
            history += "Observation: you already tried this. Try something different or finalize.\n"
        seen_actions.add(sig)

        if action not in TOOLS:
            obs = f"unknown tool '{action}'. Available: {list(TOOLS)}"
        else:
            try:
                obs = TOOLS[action](action_input)
            except Exception as e:
                obs = f"tool error: {e}"

        history += f"Observation: {obs}\n"

    return "max_steps reached without final answer"

if __name__ == "__main__":
    q = "What is the population of the country whose capital is Tokyo, divided by 1000?"
    print(run_agent(q))

§1.2评分维度 · senior 看的不是"能跑"

评分点 · 100 分制
  1. 20 分 · 基本能跑——loop / parse / tool dispatch 都对，能给出 final answer
  2. 15 分 · 安全边界——max_steps 限制 + tool 异常 try/except + 未知 tool 处理 + 重复 action 检测
  3. 15 分 · prompt 质量——明确格式约束 + Final Answer / Action 互斥 + tool 描述清楚
  4. 10 分 · stop sequence——用 stop=["\nObservation:"] 防 LLM 自己幻觉 observation（关键加分点）
  5. 10 分 · history 累积——所有历史正确传给 LLM · 不丢 step
  6. 10 分 · 代码组织——tool 是 dict 而非 if-elif 串 · 容易加新 tool
  7. 10 分 · 主动讲限制——告诉面试官"我这版还有什么 limitation 在生产要补"（streaming / parallel tool / structured output / observability）
  8. 10 分 · 演化路径——能讲"如果给我 2 小时我会再加 X / Y / Z"

§1.3常见 anti-pattern · 这些写法会扣分

Anti-pattern 1 · 没有 stop sequence

没设 stop=["\nObservation:"] → LLM 经常自己幻觉 observation 内容继续编。面试官故意用 multi-hop 题就是钓你这个。

Anti-pattern 2 · max_steps 默认无限

不设上限 → 死循环时调爆 API。必须 hard limit。

Anti-pattern 3 · tool 异常直接 raise

obs = TOOLS[action](action_input) 不 try/except → tool 报错整个 agent 崩。必须包 try 把 error 转成 observation 让 LLM 自己想办法。

Anti-pattern 4 · history 用 messages 数组而非纯文本

很多人习惯 messages=[{role, content}] 反复 append。这本身没错，但ReAct 经典实现用 single-message history更易调试 + parse · live coding 选简的。

Anti-pattern 5 · 重复 action 不检测

LLM 经常陷入"再 search 一遍同样 query"。代码里加 seen_actions set + 命中后给 LLM 一句 "you already tried this"——5 行代码救一堆面试。

§1.4面试官追问 · 5 个标准动作

  1. "为什么不用 OpenAI function calling 协议" → 答："function calling 是更现代的同类 idea。我可以 5 分钟改写。但 ReAct 的 text 格式有 2 个优势：跨 model 通用 (open model 也能跑) + thought 显式可读 / 易 trace。生产我会用 function calling 因为 schema 强 + parse 不会错"
  2. "max_steps 怎么定" → "看 task 复杂度，简单 5 难 25。但单 limit 不够，还要加 cost cap (token 累计) + time budget (墙时)"
  3. "如果 tool 是个慢 API 怎么办" → 答 async / parallel tool call / timeout / cache
  4. "怎么 eval 这个 agent" → 100 道 multi-hop golden set + LLM-as-judge + 人审 sample
  5. "上 prod 还差什么" → 列：observability (LangSmith trace) / structured logging / streaming output / prompt versioning / cost tracking / safety guardrail / multi-provider fallback
senior 收尾金句

"这版 120 行能跑通，但生产 agent 至少还要 4 件事：observability (LangSmith / Langfuse) · prompt versioning (能 rollback) · cost cap per session · structured output (用 function calling 替代 text parse)。如果给我 3 天时间我会先做这 4 个"

Question 2 · live coding

用 LangGraph 写一个带 HITL 的工单 agent

考你 LangGraph 内部理解 + interrupt 机制 · 字节 / 创业 SaaS 常见

难度 senior 限时 90-120 min 形式 live coding · 给 LangGraph 文档 面方 用 LangGraph 的中后端 / SaaS 团队

用 LangGraph 实现一个 IT 工单 agent。它能：① 解析用户工单 ② 决定 action（查 KB / 重启服务 / 重置密码 / 升级人工）③ "重启服务"和"重置密码"必须先让人类 approve 才执行。Approve 后 agent 继续；reject 则告诉用户原因。要支持 resume——用户 approve 后整个 graph 不重跑前面步骤。

§2.1参考实现 · 用 LangGraph interrupt

# ticket_agent.py
from typing import Literal, TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, END, START
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import interrupt, Command
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage, BaseMessage
from pydantic import BaseModel, Field

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# ---------- state ----------
class State(TypedDict):
    ticket: str
    messages: Annotated[list[BaseMessage], add]
    decision: str | None       # "kb" | "restart" | "reset_pwd" | "human"
    decision_reason: str | None
    result: str | None
    approved: bool | None      # 由 HITL 写入

# ---------- structured output for routing ----------
class Decision(BaseModel):
    action: Literal["kb", "restart", "reset_pwd", "human"] = Field(...)
    reason: str = Field(...)

# ---------- nodes ----------
def classify(state: State):
    """LLM 决定走哪条路."""
    msg = (
        "Classify this IT ticket into one of: kb (faq lookup), "
        "restart (restart user's service), reset_pwd (reset password), "
        "human (escalate). Output structured.\n\n"
        f"Ticket: {state['ticket']}"
    )
    out = llm.with_structured_output(Decision).invoke(msg)
    return {"decision": out.action, "decision_reason": out.reason}

def human_gate(state: State):
    """高危 action 必须 HITL."""
    user_resp = interrupt({
        "action_pending": state["decision"],
        "reason": state["decision_reason"],
        "ticket": state["ticket"],
        "prompt": "approve or reject this action",
    })
    # interrupt 等用户 resume Command(resume={"approved": True/False})
    return {"approved": user_resp.get("approved", False)}

def do_restart(state: State):
    if not state.get("approved"):
        return {"result": "REJECTED · told user we won't restart"}
    # 调真 ops API · 这里 mock
    return {"result": "service restarted successfully"}

def do_reset_pwd(state: State):
    if not state.get("approved"):
        return {"result": "REJECTED · told user we won't reset password"}
    return {"result": "password reset · sent reset email"}

def do_kb(state: State):
    # mock KB lookup
    return {"result": f"KB answer for: {state['ticket'][:60]}"}

def do_human(state: State):
    return {"result": "escalated to human agent · ticket queued"}

# ---------- routing ----------
def route_after_classify(state: State) -> str:
    decision = state["decision"]
    if decision in ("restart", "reset_pwd"):
        return "human_gate"
    if decision == "kb":
        return "do_kb"
    return "do_human"

def route_after_gate(state: State) -> str:
    return f"do_{state['decision']}"

# ---------- graph ----------
g = StateGraph(State)
g.add_node("classify", classify)
g.add_node("human_gate", human_gate)
g.add_node("do_kb", do_kb)
g.add_node("do_restart", do_restart)
g.add_node("do_reset_pwd", do_reset_pwd)
g.add_node("do_human", do_human)

g.add_edge(START, "classify")
g.add_conditional_edges("classify", route_after_classify,
    ["human_gate", "do_kb", "do_human"])
g.add_conditional_edges("human_gate", route_after_gate,
    ["do_restart", "do_reset_pwd"])
for n in ["do_kb", "do_restart", "do_reset_pwd", "do_human"]:
    g.add_edge(n, END)

# 关键: checkpointer 让 interrupt 后能 resume
checkpointer = MemorySaver()
graph = g.compile(checkpointer=checkpointer)

# ---------- usage ----------
if __name__ == "__main__":
    cfg = {"configurable": {"thread_id": "ticket-001"}}
    initial = {"ticket": "My VPN service is down, please restart it", "messages": []}

    # 第一次跑 · 应该停在 human_gate
    out = graph.invoke(initial, config=cfg)
    print("--- after first invoke ---")
    print(out)  # 看 __interrupt__ payload

    # 模拟人类 approve · resume
    out2 = graph.invoke(Command(resume={"approved": True}), config=cfg)
    print("--- after approve ---")
    print(out2["result"])

§2.2评分点

评分点 · 100 分制
  1. 20 分 · State 设计——TypedDict + Annotated reducer · 知道 messages 用 add reducer 不会被覆盖
  2. 20 分 · interrupt 用对——用了 interrupt() 而不是自造一个 if-blocking · 配 checkpointer · 知道 resume 用 Command
  3. 15 分 · structured output——routing 用 with_structured_output 而不是 prompt 让 LLM "请输出 JSON"
  4. 10 分 · routing 边界——conditional_edges 加 path list 让 graph 验证可达性
  5. 10 分 · 高危 action 隔离——只 restart / reset_pwd 走 HITL · kb 不浪费用户时间
  6. 10 分 · checkpoint 持久化——MemorySaver 是 demo，能讲生产用 PostgresSaver / RedisSaver
  7. 10 分 · reject 路径——人 reject 不能默默不做，要回 user "为什么没做"
  8. 5 分 · thread_id 概念——能讲多用户 multi-tenant 场景 thread_id 怎么生成

§2.3常见 anti-pattern

Anti-pattern 1 · 没用 interrupt · 自己 raise

有人写 raise NeedHumanApproval(...) 然后外面 try-catch 弹 UI · resume 时整个重跑。错——没用 LangGraph 提供的能力。interrupt() 是设计好的，会自动持久化 state、resume 时只跑 interrupt 之后的节点。

Anti-pattern 2 · 把 routing 逻辑塞进 LLM prompt

"LLM 你看着输出 JSON 我自己 parse" → schema 偶尔出错。用 with_structured_output(Pydantic)—— LangChain 帮你 validate + retry。

Anti-pattern 3 · MemorySaver 当生产用

MemorySaver 是 in-process · 服务重启 state 全丢。生产必须 PostgresSaver / RedisSaver。面试时用 MemorySaver 但要主动声明"生产换 X"。

Anti-pattern 4 · approve 后才检查 approved

有人在 do_restart 里没检查 state["approved"] 直接执行——这违背 HITL 全部目的。每个 destructive node 必须 verify approved。

Anti-pattern 5 · thread_id 全局共享

所有用户同 thread_id → 后面用户看到前面的 state。thread_id 必须 per session / per user，通常 f"{user_id}:{ticket_id}"。

§2.4面试官追问

  1. "如果 user 半小时不 approve 怎么办" → 设 timeout · 超时 default behavior（escalate human / cancel） · LangGraph 的 interrupt 不带 timeout 你要自己加 cron 检查 stale thread
  2. "如果 service restart 失败怎么办" → tool 异常 catch · 写到 result · 提示用户 + log incident · 不要 silently fail
  3. "扩到 1000 QPS · MemorySaver 不行了" → PostgresSaver + read replica · checkpoint 大表设 TTL · 监控 checkpoint write latency
  4. "approve 用什么 UI" → push notification · Slack interactive button · UI banner with approve/reject button + reason input
  5. "audit log 怎么存" → 每次 interrupt + resume 都 emit event 到 audit table（who / when / approved / reason）

§3.2Agent 主体 (LangGraph 版)

# src/agent.py
from langgraph.graph import StateGraph, END, START
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, ToolMessage, SystemMessage
from .tools import TOOLS, TOOL_SCHEMAS
from .prompts import SYSTEM_PROMPT

llm = ChatOpenAI(model="gpt-4o", temperature=0).bind_tools(TOOL_SCHEMAS)

def agent_node(state):
    msgs = [SystemMessage(content=SYSTEM_PROMPT)] + state["messages"]
    resp = llm.invoke(msgs)
    return {"messages": [resp]}

def tool_node(state):
    last = state["messages"][-1]
    out = []
    for tc in last.tool_calls:
        name = tc["name"]
        try:
            result = TOOLS[name](**tc["args"])
        except Exception as e:
            result = f"tool error: {e}"
        out.append(ToolMessage(content=str(result), tool_call_id=tc["id"]))
    return {"messages": out}

def should_continue(state) -> str:
    last = state["messages"][-1]
    if last.tool_calls:
        return "tools"
    return END

g = StateGraph(dict)
g.add_node("agent", agent_node)
g.add_node("tools", tool_node)
g.add_edge(START, "agent")
g.add_conditional_edges("agent", should_continue, ["tools", END])
g.add_edge("tools", "agent")

# 关键 · recursion_limit + checkpointer 都要
agent = g.compile()

def run(question: str, max_iters: int = 15) -> dict:
    init = {"messages": [HumanMessage(content=question)]}
    out = agent.invoke(init, {"recursion_limit": max_iters})
    final = out["messages"][-1].content
    # 提取 trace · 给 eval 用
    tool_calls = [m for m in out["messages"] if hasattr(m, "tool_calls") and m.tool_calls]
    return {"answer": final, "trace": out["messages"], "tool_calls": len(tool_calls)}

§3.3Hybrid Retrieval (高加分)

# src/retrieve.py
from rank_bm25 import BM25Okapi
import chromadb
from sentence_transformers import CrossEncoder

class HybridRetriever:
    def __init__(self, chunks):
        self.chunks = chunks
        # BM25
        self.bm25 = BM25Okapi([c["text"].split() for c in chunks])
        # Vector
        self.client = chromadb.Client()
        self.col = self.client.create_collection("docs")
        self.col.add(
            documents=[c["text"] for c in chunks],
            ids=[c["id"] for c in chunks],
            metadatas=[{"file": c["file"]} for c in chunks],
        )
        # Reranker
        self.reranker = CrossEncoder("BAAI/bge-reranker-base")

    def search(self, query: str, k: int = 10) -> list[dict]:
        # BM25 top 30
        bm25_scores = self.bm25.get_scores(query.split())
        bm25_top = sorted(range(len(bm25_scores)), key=lambda i: -bm25_scores[i])[:30]
        # Vector top 30
        vec_results = self.col.query(query_texts=[query], n_results=30)
        vec_top = [int(i) for i in vec_results["ids"][0]]
        # Union
        candidate_ids = list(set(bm25_top + vec_top))
        # Rerank
        pairs = [[query, self.chunks[i]["text"]] for i in candidate_ids]
        scores = self.reranker.predict(pairs)
        ranked = sorted(zip(candidate_ids, scores), key=lambda x: -x[1])[:k]
        return [self.chunks[i] for i, _ in ranked]

§3.4Eval script (必须给)

# src/eval.py
import json
from .agent import run

def judge(predicted: str, ground_truth: str, question: str) -> bool:
    """LLM-as-judge · 看预测是否含 ground truth 关键 fact."""
    from openai import OpenAI
    client = OpenAI()
    msg = f"""Question: {question}
Ground truth: {ground_truth}
Predicted answer: {predicted}

Does the predicted answer contain the key facts from ground truth? Reply only 'YES' or 'NO'."""
    resp = client.chat.completions.create(model="gpt-4o-mini",
        messages=[{"role": "user", "content": msg}], temperature=0)
    return "YES" in resp.choices[0].message.content.upper()

def main():
    samples = [json.loads(l) for l in open("eval/sample.jsonl")]
    results = []
    for s in samples:
        out = run(s["question"])
        passed = judge(out["answer"], s["answer"], s["question"])
        results.append({**s, "predicted": out["answer"],
                       "passed": passed, "tool_calls": out["tool_calls"]})
    success = sum(1 for r in results if r["passed"]) / len(results)
    print(f"Success rate: {success:.1%} ({sum(1 for r in results if r['passed'])}/{len(results)})")
    json.dump(results, open("eval/results.json", "w"), indent=2)

    return {"answer": llm.invoke(msg).content}

def route(state) -> str:
    if "select" in state["qtype"]:
        return "execute"
    return "execute"  # write 也直接执行

§4.24 个 bug 的位置和修复

Bug 1 · 死循环 · "复杂 query 跑不停"

位置：根本没看出来——题目说"死循环"是陷阱。这个 graph 是 DAG 没循环根本死不了。 真因：是 execute 步骤被外部 service 卡住，没 timeout。但还有更隐蔽的：temperature=0.7 让 SQL gen 不稳定 + 没 retry → 用户重复问 = 跑很多遍 = 看起来"死循环"。

修复：

  • temperature=0 让 SQL 确定性
  • execute 加 timeout (e.g. 30s)
  • recursion_limit + 复杂 query 拒绝（dry-run cost 估计）
Bug 2 · 答非所问 · "偶尔结果跟问题不对"

位置：gen_sql 节点 · history 累积所有 prompt 文本 + LLM 输出 · 越来越长 + 把过去的 question 和 SQL 都混进 prompt。LLM 经常被旧 question 影响生成跟当前 question 不匹配的 SQL。 这是 senior 必看出来的 bug——history 累积错地方。

修复：

  • 不要把 history 拼成大 string · 用 messages 数组让 LLM 知道哪是 user 哪是 assistant
  • 每个 question 是新的 invocation · 不带 history（除非真要 multi-turn）
  • 如果要 multi-turn，用 LangGraph state 的 messages reducer 而不是手动拼字符串
Bug 3 · 写操作直接执行 · 没 HITL

位置：route 函数 · 看似有 if-else 但两个 branch 都返回 "execute"——写 SQL 跟读 SQL 走同条路。注释 "write 也直接执行" 是凶手。

修复：

def route(state) -> str:
    if "select" in state["qtype"]:
        return "execute"
    return "human_gate"  # write 走 HITL

加 human_gate 节点用 interrupt() · approved 才进 execute · 否则告诉用户 reason。

Bug 4 · memory 串号 · "用户 A 看到用户 B 的历史"

位置：模块底部 THREAD = {"configurable": {"thread_id": "sql-agent"}}——所有用户共享同一 thread_id。配 MemorySaver 后 state 持久跨用户混。

修复：

def ask(question: str, user_id: str, session_id: str) -> str:
    cfg = {"configurable": {"thread_id": f"{user_id}:{session_id}"}}
    out = graph.invoke({"question": question}, cfg)
    return out["answer"]

# multi_agent.py
from typing import Literal, Annotated, TypedDict
from operator import add
from langgraph.graph import StateGraph, END, START
from langchain_openai import ChatOpenAI
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage, SystemMessage
from pydantic import BaseModel, Field

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

class State(TypedDict):
    messages: Annotated[list[BaseMessage], add]
    research_done: bool
    code_done: bool
    writer_done: bool
    iterations: int  # 防死循环

class RouterDecision(BaseModel):
    next_agent: Literal["research_agent", "code_agent", "writer_agent", "FINISH"]
    reason: str

SUPERVISOR_PROMPT = """You are the supervisor of 3 specialist agents.
- research_agent: gathers web info
- code_agent: writes code examples
- writer_agent: produces the final user-facing output

Decide next agent based on state. FINISH when writer_agent has produced output.
Don't repeat agents unless their work was inadequate."""

def supervisor(state: State):
    # 关键 · 显式跟踪每 agent 是否做过
    status = (
        f"research_done={state.get('research_done', False)}, "
        f"code_done={state.get('code_done', False)}, "
        f"writer_done={state.get('writer_done', False)}"
    )
    msg = SUPERVISOR_PROMPT + f"\n\nCurrent status: {status}\nMessages so far: {len(state['messages'])}"
    out = llm.with_structured_output(RouterDecision).invoke(
        [SystemMessage(content=msg), *state["messages"][-5:]]
    )
    print(f"[supervisor] → {out.next_agent} · {out.reason}")
    return {"messages": [AIMessage(content=f"[supervisor]: route to {out.next_agent} ({out.reason})",
                                       name="supervisor")],
            "iterations": state.get("iterations", 0) + 1}

def research_agent(state: State):
    msg = ("You are a researcher. Read the user request and provide 3-5 key facts. "
           "Mock web search · pretend you searched.")
    out = llm.invoke([SystemMessage(content=msg), *state["messages"]])
    out.name = "research_agent"
    return {"messages": [out], "research_done": True}

def code_agent(state: State):
    msg = "You write code examples. Use the research findings if available."
    out = llm.invoke([SystemMessage(content=msg), *state["messages"]])
    out.name = "code_agent"
    return {"messages": [out], "code_done": True}

def writer_agent(state: State):
    msg = "You produce the FINAL user-facing answer. Combine research + code into a clean response."
    out = llm.invoke([SystemMessage(content=msg), *state["messages"]])
    out.name = "writer_agent"
    return {"messages": [out], "writer_done": True}

def route_supervisor(state: State) -> str:
    # 防死循环
    if state.get("iterations", 0) >= 8:
        return END
    last = state["messages"][-1]
    text = last.content
    if "FINISH" in text:
        return END
    if "research_agent" in text:
        return "research_agent"
    if "code_agent" in text:
        return "code_agent"
    if "writer_agent" in text:
        return "writer_agent"
    return END

g = StateGraph(State)
g.add_node("supervisor", supervisor)
g.add_node("research_agent", research_agent)
g.add_node("code_agent", code_agent)
g.add_node("writer_agent", writer_agent)
g.add_edge(START, "supervisor")
g.add_conditional_edges("supervisor", route_supervisor,
    ["research_agent", "code_agent", "writer_agent", END])
# 每个 specialist 跑完回 supervisor
for n in ["research_agent", "code_agent", "writer_agent"]:
    g.add_edge(n, "supervisor")

graph = g.compile()

if __name__ == "__main__":
    out = graph.invoke({"messages": [HumanMessage(
        content="Write a 200-word explanation of PagedAttention with a code example")],
        "iterations": 0}, {"recursion_limit": 20})
    print("\n\nFINAL:\n", out["messages"][-1].content)

# app.py · FastAPI + LangGraph streaming
from fastapi import FastAPI, Request
from sse_starlette.sse import EventSourceResponse
import asyncio
from langgraph.checkpoint.postgres import PostgresSaver  # 生产用 Postgres 不 Memory
from langchain_core.messages import HumanMessage
from .agent_graph import g  # 已有的 graph

app = FastAPI()
checkpointer = PostgresSaver.from_conn_string("postgresql://...")
checkpointer.setup()
agent = g.compile(checkpointer=checkpointer)

# 全局取消管理
ACTIVE_RUNS: dict[str, asyncio.Event] = {}  # thread_id → cancel event

@app.post("/chat")
async def chat(req: Request, body: dict):
    user_id = body["user_id"]
    session_id = body["session_id"]
    question = body["question"]
    thread_id = f"{user_id}:{session_id}"

    cancel = asyncio.Event()
    ACTIVE_RUNS[thread_id] = cancel

    cfg = {"configurable": {"thread_id": thread_id}}

    async def event_gen():
        try:
            # 关键 1: astream_events 拿 LLM token 级 stream
            async for event in agent.astream_events(
                {"messages": [HumanMessage(content=question)]},
                cfg, version="v2"
            ):
                # 关键 2: 检查取消信号
                if cancel.is_set():
                    yield {"event": "cancelled", "data": "user stopped"}
                    return

                # 关键 3: 选 token-level event · 不是 node-level
                if event["event"] == "on_chat_model_stream":
                    chunk = event["data"]["chunk"]
                    if chunk.content:
                        yield {"event": "token", "data": chunk.content}

                elif event["event"] == "on_chain_end" and event["name"] == "LangGraph":
                    yield {"event": "done", "data": ""}
        finally:
            ACTIVE_RUNS.pop(thread_id, None)

    return EventSourceResponse(event_gen())

@app.post("/stop")
async def stop(body: dict):
    thread_id = f"{body['user_id']}:{body['session_id']}"
    if thread_id in ACTIVE_RUNS:
        ACTIVE_RUNS[thread_id].set()
        return {"stopped": True}
    return {"stopped": False}

@app.post("/resume")
async def resume(body: dict):
    """服务重启 · 用户继续聊 · checkpointer 自动恢复 state."""
    thread_id = f"{body['user_id']}:{body['session_id']}"
    cfg = {"configurable": {"thread_id": thread_id}}
    state = agent.get_state(cfg)
    return {"last_messages": [m.content for m in state.values["messages"][-3:]],
            "can_resume": state.next != ()}

评分点
  1. 20 分——用 astream_events 拿 token 级（不是 astream 拿 node 级 chunk · 那是 node 完才 emit）
  2. 15 分——SSE / WebSocket 真 streaming（不是攒一坨再发）
  3. 15 分——cancel event 机制让 stop 能立即生效（不是等 LLM 跑完）
  4. 15 分——PostgresSaver / RedisSaver 不是 MemorySaver（生产 must）
  5. 10 分——thread_id = user:session 隔离
  6. 10 分——/resume 用 get_state 复活上次 state
  7. 10 分——能讲 streaming 失败/中断时 partial response 怎么处理
  8. 5 分——主动讲 backpressure / connection drop / token cost mid-stream 等 edge case
最常错 · stream 错对象

很多人用 agent.astream(state) 以为是 token streaming —— 错！那是 node-level streaming（每 node 跑完 emit 一次）。 token-level 必须用 astream_events(version="v2") + filter on_chat_model_stream。

# eval_harness.py
import json, time, asyncio, statistics
from dataclasses import dataclass, asdict
from typing import Literal
from openai import OpenAI
from .agent import run_agent  # 给定的 agent

judge_llm = OpenAI()

@dataclass
class EvalCase:
    id: str
    question: str
    expected: str
    category: str  # "billing" | "technical" | ...

@dataclass
class EvalResult:
    case_id: str
    answer: str
    passed: bool
    failure_kind: str | None  # "halluc" | "wrong_tool" | "no_answer" | "timeout" | None
    latency_ms: int
    cost_usd: float
    tool_calls: int

def classify_failure(case: EvalCase, result: dict) -> str:
    if result.get("timeout"): return "timeout"
    if not result.get("answer"): return "no_answer"
    # LLM judge 失败原因
    msg = f"""Question: {case.question}
Expected: {case.expected}
Actual: {result['answer']}
Tool calls: {result.get('tool_calls', [])}

Why did this fail? Reply only one of: halluc / wrong_tool / no_answer / partial / other"""
    out = judge_llm.chat.completions.create(model="gpt-4o-mini",
        messages=[{"role": "user", "content": msg}], temperature=0)
    return out.choices[0].message.content.strip().split()[0]

def judge_pass(case: EvalCase, answer: str) -> bool:
    # LLM-as-judge with rubric
    msg = f"""Question: {case.question}
Expected key facts: {case.expected}
Actual answer: {answer}

Does the actual answer cover the expected key facts? Reply YES or NO only."""
    out = judge_llm.chat.completions.create(model="gpt-4o-mini",
        messages=[{"role": "user", "content": msg}], temperature=0)
    return "YES" in out.choices[0].message.content.upper()

def run_one(case: EvalCase) -> EvalResult:
    t0 = time.time()
    try:
        result = run_agent(case.question, timeout=30)
    except TimeoutError:
        result = {"answer": "", "timeout": True, "cost": 0, "tool_calls": []}
    latency = int((time.time() - t0) * 1000)
    answer = result.get("answer", "")
    passed = answer and judge_pass(case, answer)
    failure = None if passed else classify_failure(case, result)
    return EvalResult(case.id, answer, passed, failure, latency,
                      result.get("cost", 0), len(result.get("tool_calls", [])))

def main(golden_path: str = "eval/golden.jsonl", baseline_path: str | None = None):
    cases = [EvalCase(**json.loads(l)) for l in open(golden_path)]
    results = [run_one(c) for c in cases]

    n = len(results)
    summary = {
        "total": n,
        "pass_rate": sum(1 for r in results if r.passed) / n,
        "latency_p50": statistics.median(r.latency_ms for r in results),
        "latency_p95": statistics.quantiles([r.latency_ms for r in results], n=20)[-1],
        "avg_cost": sum(r.cost_usd for r in results) / n,
        "avg_tool_calls": sum(r.tool_calls for r in results) / n,
    }

    # failure 分类
    failures = [r for r in results if not r.passed]
    by_kind: dict[str, int] = {}
    for r in failures:
        by_kind[r.failure_kind or "unknown"] = by_kind.get(r.failure_kind or "unknown", 0) + 1
    summary["failure_breakdown"] = by_kind

    # Regression check
    if baseline_path:
        baseline = json.load(open(baseline_path))
        regression = {
            "pass_rate_delta": summary["pass_rate"] - baseline["pass_rate"],
            "latency_p95_delta": summary["latency_p95"] - baseline["latency_p95"],
            "cost_delta": summary["avg_cost"] - baseline["avg_cost"],
        }
        summary["regression"] = regression
        # Fail CI if regression bad
        if regression["pass_rate_delta"] < -0.05:
            print("❌ REGRESSION: pass rate dropped > 5%")
            exit(1)

    json.dump(summary, open("eval/summary.json", "w"), indent=2)
    json.dump([asdict(r) for r in results], open("eval/results.json", "w"), indent=2)

    render_html(summary, results)
    print(json.dumps(summary, indent=2))

def render_html(summary, results):
    # 简单 HTML report · 实战可用 jinja2 · 这里 inline
    rows = "".join(
        f'<tr class="{("p" if r.passed else "f")}">'
        f"<td>{r.case_id}</td><td>{r.passed}</td>"
        f"<td>{r.failure_kind or '-'}</td><td>{r.latency_ms}ms</td>"
        f"<td>${r.cost_usd:.4f}</td><td>{r.answer[:80]}</td></tr>"
        for r in results
    )
    html = f"""<html><style>.p{{background:#dfd}}.f{{background:#fdd}}td{{padding:4px 8px}}</style>
<h1>Eval Report</h1><pre>{json.dumps(summary, indent=2)}</pre>
<table><tr><th>ID</th><th>Pass</th><th>Failure</th><th>Latency</th><th>Cost</th><th>Answer</th></tr>
{rows}</table></html>"""
    open("eval/report.html", "w").write(html)

if __name__ == "__main__":
    main(baseline_path="eval/baseline.json")

从公开 paper 实现一个 agent 算法

staff 级开放题 · 看你的 paper-to-code 能力

难度 staff 限时 60-90 min · 或 take-home 1 周 形式 选 paper · 实现核心 idea · 跑 demo 面方 Anthropic / OpenAI / 高 bar 创业

从 Reflexion / Self-Discover / Tree of Thoughts / Voyager / CRAG 中选一篇，用 ~200-400 行实现核心 idea。要求：① 有 baseline 对照 ② 至少 1 个 task 跑出 paper claim 的方向性效果（不需要 SOTA） ③ README 讲清你做了什么 / 简化了什么 / 为什么。

§8.1怎么选 paper · 4 个挑选标准

  1. 核心 idea 简单——一段话能讲清的。Reflexion / Self-Discover 都很合适
  2. 不要 GPU 训练——只用 LLM API 能复现的（不要选要 fine-tune 的）
  3. 有公开 task——能用 HumanEval / GSM8K / HotpotQA 这种现成 benchmark
  4. 有 reference impl——读他们 GitHub 帮你少踩坑（不是抄）

推荐选 Reflexion——核心 idea 一段话能讲清、能在 1h 内 demo 出方向性效果、HumanEval 上能看到提升。

§8.2Reflexion 简化实现 (~150 行)

# reflexion.py · 简化版
from openai import OpenAI
from dataclasses import dataclass, field
import subprocess, tempfile, os

llm = OpenAI()

@dataclass
class Trial:
    code: str
    test_output: str
    passed: bool
    reflection: str = ""

def generate_code(prompt: str, reflections: list[str]) -> str:
    refl_text = ""
    if reflections:
        refl_text = "\n\nLessons from previous failed attempts:\n- " + "\n- ".join(reflections)

    msg = f"""Write Python code to solve:
{prompt}

{refl_text}

Output ONLY the function body, no explanation."""
    out = llm.chat.completions.create(model="gpt-4o-mini", temperature=0.2,
        messages=[{"role": "user", "content": msg}])
    return out.choices[0].message.content

def run_tests(code: str, tests: str) -> tuple[bool, str]:
    full = code + "\n\n" + tests
    with tempfile.NamedTemporaryFile(mode="w", suffix=".py", delete=False) as f:
        f.write(full)
        path = f.name
    try:
        proc = subprocess.run(["python", path], capture_output=True, text=True, timeout=10)
        passed = proc.returncode == 0 and "FAIL" not in proc.stdout
        return passed, proc.stdout + proc.stderr
    finally:
        os.unlink(path)

def reflect(prompt: str, code: str, error: str) -> str:
    """核心 · 把失败转成下一次的 lesson."""
    msg = f"""Task: {prompt}
Your code:
{code}

It failed with:
{error[:500]}

Write ONE concrete lesson in 1 sentence (e.g. "Remember to handle empty list edge case")."""
    out = llm.chat.completions.create(model="gpt-4o-mini", temperature=0.5,
        messages=[{"role": "user", "content": msg}])
    return out.choices[0].message.content.strip()

def solve(prompt: str, tests: str, max_trials: int = 3) -> tuple[bool, list[Trial]]:
    reflections: list[str] = []
    trials: list[Trial] = []
    for i in range(max_trials):
        code = generate_code(prompt, reflections)
        passed, output = run_tests(code, tests)
        trial = Trial(code=code, test_output=output, passed=passed)
        if passed:
            trials.append(trial)
            return True, trials
        # 失败 · reflect
        lesson = reflect(prompt, code, output)
        trial.reflection = lesson
        trials.append(trial)
        reflections.append(lesson)
    return False, trials

# --- Demo on a HumanEval-style problem ---
if __name__ == "__main__":
    prompt = "Write a function `def has_close_pair(nums, threshold)` returning True if any two distinct elements are within threshold of each other."
    tests = """
assert has_close_pair([1.0, 2.0, 3.0], 0.5) == False
assert has_close_pair([1.0, 2.8, 3.0, 4.0], 0.3) == True
assert has_close_pair([], 1.0) == False
assert has_close_pair([1.0], 1.0) == False
print("PASS")
"""

    # Baseline: 单次生成
    bl_code = generate_code(prompt, [])
    bl_passed, bl_out = run_tests(bl_code, tests)
    print(f"Baseline (1 try): {'PASS' if bl_passed else 'FAIL'}")

    # Reflexion: 多 trial 带 reflection
    rfl_passed, trials = solve(prompt, tests, max_trials=3)
    print(f"Reflexion: {'PASS' if rfl_passed else 'FAIL'} after {len(trials)} trials")
    for i, t in enumerate(trials):
        print(f"  Trial {i+1}: {'pass' if t.passed else 'fail'} · lesson: {t.reflection}")

附录 A · 反例

22 个 senior 面试 anti-pattern · 这些写法直接扣分

前面每题都列了具体 anti · 这里再 cross-cut 总结

架构层 (8)

  1. 上来就 LangGraph——简单题 100 行能写完，硬上框架显得不会基本功
  2. 无脑 multi-agent——能 single 就 single · multi-agent 必须能讲为什么不能 single
  3. 同步阻塞调 LLM——senior 默认 async·至少要能讲 streaming + parallel tool
  4. 没 max_iterations / recursion_limit——agent 必须有上限
  5. tool 是 if-elif 串——tool 应该是 dict / registry · 易扩展
  6. 所有 tool 串行调——独立 tool 必须 parallel
  7. memory 拼成大 string——用 messages 数组 · LLM 才知道角色
  8. routing 用 text parse——用 structured output (Pydantic / function calling)

正确性层 (7)

  1. tool 异常不 catch——agent 直接崩 · 应转成 observation 让 LLM 处理
  2. 没 stop sequence——LLM 自己幻觉 observation 继续编
  3. temperature 默认 0.7——agent 推理任务用 0 · routing 必须 0
  4. system prompt 没说 "Final Answer 唯一"——LLM 经常多个 final 同时给
  5. HITL 用 raise 实现——用框架的 interrupt · 才能 resume 不重跑
  6. thread_id 全局共享——必须 per-user / per-session
  7. writer 写完不验证——final output 该 LLM-as-judge 一道

生产层 (7)

  1. MemorySaver 当生产——必须 Postgres / Redis
  2. 没 cost cap——一个 abuse user 能花你 $1000+
  3. 没 timeout——tool / LLM call 都要 timeout
  4. 没 trace / metric——上线后 debug 全靠 print log
  5. prompt 写死代码里——应该 versioned · 能 rollback · 关联 eval
  6. 没 multi-provider fallback——OpenAI 挂 30min 你的服务也挂 30min
  7. 没 eval baseline——改 prompt 后不知道是变好还是变坏
